## 의도

페이지를 git 형상관리처럼 다룬다. 작성 중인 상태와 외부에 보여줄 확정 상태를 분리하고, 버전 히스토리·diff·공개 링크 버전 선택을 모두 1급으로 둔다. Manadocs 가 "초안만 떠있는 위키" 가 아니라 **"버전 인식하는 협업 채널"** 로 진화하기 위한 토대.

## 핵심 아이디어

- **모든 페이지는 버전 0(페이지 생성) 에서 시작**. 명시적 `문서확정` 으로 버전 1, 2, … 가 쌓인다.
- **작업문서(working) ↔ 버전(committed) 분리** — 작업문서는 작성자/공동작업자가 갱신, 독자는 Primary 버전을 본다.
- **작업문서는 다중 가능** — 한 페이지에 여러 작업문서가 동시에 존재. 그중 하나가 Primary 위치를 가지며 나머지는 보조. 각 작업문서는 특정 과거 버전에서 시작(`버전 N에서 시작`).
- **Primary 는 가변** — 자동으로 마지막 확정 버전이 Primary 가 아니다. 사용자가 버전 항목의 ⋯ 메뉴에서 "Primary 로 변경" 으로 swap 한다. Primary 가 외부 공개 기본 대상.
- **문서확정(commit)** — 작업문서를 새 버전으로 확정. 메시지 + 작성자 + 참여자 기록.
- **푸터 도구막대** — 페이지 본문 하단 floating pill. 이모지 + 문서버전 + diff stats(+/-) + DIFF / 수정취소 / 문서확정.
- **공유** — 페이지 단위. 두 가지 링크 종류를 동시에 발급 가능:
  - 최신 Primary 공개 (Primary 가 바뀌면 자동 반영)
  - 특정 버전 고정 공개 (그 버전 스냅샷만 보여줌)
- **미리보기 모달** — 임의 버전을 reader 시점으로 미리 봄. 페이지 툴바의 미리보기 버튼.
- **이모지** — 페이지마다 이모지 지정 가능. 미지정이면 기본 문서(📄). 사이드바 트리·breadcrumb·푸터 pill 모두 같은 이모지를 노출. 이모지 클릭으로 변경(픽커). 기존에 MCP 로만 가능했던 기능을 UI 1급으로 노출.

## 삭제·폐기 정책

- **작업문서**: "삭제" — 폐기 개념 없음. 안 쓰면 삭제, 끝.
- **버전(문서)**: "폐기" 존재. 폐기된 버전은 외부 공개에서 사라짐.
  - 그 버전을 가리키는 특정버전 고정 공유 링크는 **fallback** 또는 **404** 정책 둘 중 선택.
    - fallback: 가장 가까운 비폐기 버전(이전 또는 다음)으로 자동 전환
    - 404: 명시적으로 끊는다
  - UI: 버전 항목에 폐기됨 배지 + 메시지 strikethrough.
- 폐기 정책 기본값과 페이지·링크 단위 override 정책은 별도 결정.

## 컨텍스트 메뉴 (버전 항목 ⋯)

- **Primary 로 변경** — 현재 Primary 자리에서 이 버전을 Primary 로 swap
- **문서 Duplicate** — 그 버전의 페이지 스냅샷을 베이스로 **새 페이지** 생성. 트리 dependency 만 carry, 버전 체인은 carry 하지 않음(새 페이지는 자기 버전 0 으로 시작).
- (확장 후보) 폐기 / 폐기 해제 / 다운로드(JSON·MD) / 비교

## 리뷰

- **리뷰는 버전(committed) 에만** — 작업문서에는 리뷰 불가. (작업문서는 임시 변경 단위라 리뷰 대상 부적합)
- 리뷰 패널 UX:
  - 기본: 현재 Primary 버전의 리뷰만 표시
  - 체크박스 "모든 리뷰 보기" 켜면 다른 버전의 리뷰도 함께 (버전 단위 그룹핑)

## 작업문서 참여자

- 작업문서 카드는 변경에 참여하거나 리뷰한 모든 사람을 아바타 스택으로 표시.
- 작업문서 자체엔 리뷰가 없지만, 그 작업문서가 확정된 뒤 만들어질 버전에 대한 리뷰가 들어올 수 있다는 점은 별개.

## UI 변경 (이번 시안 반영)

- 페이지 툴바 좌측에 탭: 작업문서 / 버전 / 리뷰
- 페이지 툴바 우측: Share(카운트 칩) + 미리보기 (Edit/Read 토글 제거)
- 페이지 본문 상단에 breadcrumb (이모지 포함)
- 사이드바 트리: 페이지마다 이모지 (미지정 시 기본 문서 이모지)
- 페이지 본문 하단 floating pill: 이모지 + 문서버전 + diff stats + DIFF / 수정취소 / 문서확정
- 우측 도킹 패널 3종 (탭에 따라 토글): 작업문서 / 버전 / 리뷰
- 모달 2종: 미리보기 / 공유

## 작업 순서 메모 (B/A 와의 관계)

이 항목이 **가장 침습적** (DB 스키마·페이지 모델·에디터 working-copy·공유 링크 모두 건드림). 다른 항목보다 먼저 박는 게 retrofit 비용을 가장 줄인다.

분할 옵션:
- **C′ (얇은 1차)**: 스냅샷 히스토리 read-only 만. commit/history 까지.
- **C 잔여**: working-copy 분리 · diff 뷰 · 공개 링크 버전 선택 · 폐기 정책.

C′ 만으로도 DB 가 "현재 버전 + 스냅샷 N개" 모델이 잡혀, Artifacts(B) 가 "특정 페이지 버전에서 임베드" 라는 의미를 가질 토대가 됨.

## 미확정·상의 대상

- 폐기 정책 기본값 (fallback vs 404) 및 페이지·링크 단위 override 허용 여부
- 작업문서 다중 시 동시 편집 시 충돌 해결 (각 작업문서는 독립 working copy 인가, 공유 working copy 인가)
- Primary 변경 시 외부 공개 링크 자동 업데이트 정책 (즉시 반영 vs 사용자 확인 후 반영)
- 문서 Duplicate 시 carry 할 메타 범위 (스페이스 소속 / 권한 / 첨부 / 이모지 — 전부 vs 일부)
- 리뷰 패널의 "모든 리뷰 보기" 시 그룹핑·정렬 (버전 내림차순? 최근 리뷰 우선?)
- 이모지 픽커 구현 — 자체 vs 라이브러리 (`emoji-mart` 등)
- 리뷰 자체의 데이터 모델 (현재 Manadocs 리뷰 기능과의 호환·이관)
- Hocuspocus 실시간 동기화와의 충돌 (working 상태가 실시간 공유돼야 하는가, 작성자 단독인가)
- 마이그레이션 전략 (기존 페이지 → 버전 0 + 현재 본문을 버전 1 로 잡기)
